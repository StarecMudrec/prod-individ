# Архитектура приложения «Твоя игра»

## Обзор

Одностраничное приложение (SPA) на Vue 3. Игровая логика и состояние хранятся на клиенте; серверная часть не используется. Состояние можно сохранять в `localStorage` для восстановления после перезагрузки страницы.

## Структура каталогов

```
├── public/                 # Результат сборки (npm run build)
├── static/                 # Статические файлы (копируются в public при сборке)
├── src/
│   ├── components/         # Переиспользуемые компоненты
│   │   ├── ui/             # Кнопки, карточки, общие элементы интерфейса
│   │   └── game/           # Игровые компоненты (табло, карточка игрока и т.д.)
│   ├── composables/        # Композиции Vue (useLocalStorage, таймеры и т.д.)
│   ├── lib/                # Утилиты и константы (без привязки к Vue)
│   ├── router/             # Маршруты Vue Router
│   ├── stores/             # Сторы Pinia (игровое состояние)
│   ├── styles/             # Глобальные стили и дизайн-токены
│   ├── types/              # Типы TypeScript (игрок, вопрос, раунд и т.д.)
│   ├── views/              # Страницы/экраны (главная, настройка, табло, вопрос)
│   ├── App.vue
│   └── main.ts
├── index.html
├── vite.config.ts
├── tailwind.config.js
└── package.json
```

## Роутинг

| Путь       | Имя      | Назначение                    |
|-----------|----------|-------------------------------|
| `/`       | home     | Главная, кнопка «Начать игру» |
| `/setup`  | setup    | Ввод имён и клавиш игроков    |
| `/board`  | board    | Игровое табло (темы и цены)   |
| `/question` | question | Экран вопроса с таймером и вводом ответа |

Переходы между экранами выполняются через `router.push()` в зависимости от фазы игры.

## Состояние (Pinia)

- **game** — игроки, текущая фаза (`setup` | `round-board` | `question` | `round-summary` | `final` | `results`), активный игрок, кто выбирает вопрос, номер раунда.

Дополнительные сторы при необходимости: пакеты тем/вопросов, настройки (dev-mode, звук).

## Дизайн-система

- Переменные в `src/styles/variables.css` (цвета, шрифты, отступы, радиусы).
- В Tailwind они подключены в `tailwind.config.js` (theme.extend).
- Компоненты используют классы Tailwind и при необходимости ссылаются на переменные.

## Взаимодействие частей

1. **Роутер** отображает одну из **views** в зависимости от URL.
2. **Views** используют **components** (ui и game) и читают/изменяют **stores**.
3. **Composables** инкапсулируют логику (localStorage, таймеры, клавиатура).
4. **types** задают контракты данных для сторов, вопросов и игроков.
5. **lib** содержит константы (стоимости вопросов, таймауты) и чистые функции.

Рекомендуется держать файлы до ~150 строк, выносить сложную логику в composables или lib и избегать дублирования кода.
